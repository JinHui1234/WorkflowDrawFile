## 1. 概述

### 1.1 目的

本文档定义MES上位机（MASTER）与PLC下位机之间的标准通信接口协议，包括：
- 数据块地址定义
- 信号语义与操作规则
- 握手流程与时序要求
- 异常处理与超时规范

本协议确保MES系统与PLC设备之间的可靠数据交换，支持产品追溯、过程控制、质量管理等核心制造功能。

### 1.2 适用范围

本协议适用于以下场景：
- 产品上线/首件/返修/转线
- 工位作业许可控制
- 工艺过程质量数据采集
- 设备状态监控

### 1.3 协议设计原则

1. **握手确认原则：** 所有关键信号均需对方应答确认，不允许单边操作
2. **先写后读原则：** 数据先写入DB块，再置位对应触发信号
3. **复位对称原则：** 谁置位的信号由对方复位触发，确保逻辑闭环
4. **防呆原则：** MES须校验所有产品信息，错误产品不允许作业
5. **超时保护原则：** 所有等待操作均设置超时，避免系统死锁

## 3. 通信基础配置

### 3.1 通信方式

| 项目 | 规范 |
|---|---|
| 协议类型 | Siemens S7 TCP/IP（推荐）或 OPC-UA / Modbus TCP |
| PLC IP地址 | 由现场电气工程师提供（建议固定IP） |
| MES IP地址 | 由IT/网络工程师分配 |
| 通信周期 | ≤ 100ms（建议50ms） |
| 数据编码 | 西门子Big-Endian字节序 |
| 字符编码 | ASCII（SN等字符串字段） |

### 3.2 数据块规划

| DB块 | 方向 | 说明 | 大小建议 |
|---|---|---|---|
| DB201 | MASTER → PLC | MES向PLC写入的控制/数据区 | 256 Bytes |
| DB200 | PLC → MASTER | PLC向MES上报的状态/数据区（推荐） | 256 Bytes |

> ⚠️ **注意：** DB200（PLC→MASTER方向）为推荐补充，需与PLC工程师确认实际地址。

| Byte 48~49 | 2 | 异常码 |

| Byte 56~255 | 200 | 工位质量数据（最多10组） |

## 5. MASTER→PLC 信号详细定义（DB201）

### 5.1 Byte 0 — 主控制位字节

**PLC引用地址：** DB201.DBB0  

| 位地址 | 信号名称 | 数据类型 | 置位方 | 复位方 | 说明 |
|---|---|---|---|---|---|
| DB201.DBX0.0 | 心跳 | BOOL | MASTER | MASTER | 每1秒翻转一次；PLC监测，超过10秒无变化则通信异常报警 |
| DB201.DBX0.1 | 初始化完成 | BOOL | MASTER | MASTER | MES系统就绪可执行新流程时置True；流程开始前由MES管理 |
| DB201.DBX0.2 | 上位机回复到位信号 | BOOL | MASTER | PLC | MES收到PLC产品到位信号后置True；PLC读取后在合适时机触发MES复位 |
| DB201.DBX0.3 | 是否作业信号 | BOOL | MASTER | PLC | True=MES允许PLC执行工艺；False=MES禁止作业 |
| DB201.DBX0.4 | 上位机回复完成信号 | BOOL | MASTER | PLC | MES处理完成数据并上传后置True；PLC读取后触发MES复位 |
| DB201.DBX0.5 | 预留 | BOOL | — | — | 预留，保持False |
| DB201.DBX0.6 | 预留 | BOOL | — | — | 预留，保持False |
| DB201.DBX0.7 | 预留 | BOOL | — | — | 预留，保持False |

### 5.2 Byte 1 — 工位2扩展控制位字节

**PLC引用地址：** DB201.DBB1  

| 位地址 | 信号名称 | 数据类型 | 置位方 | 复位方 | 说明 |
|---|---|---|---|---|---|
| DB201.DBX1.0 | 上位机回复到位信号2 | BOOL | MASTER | PLC | 第二工位到位应答，与DBX0.2逻辑相同 |
| DB201.DBX1.1 | 是否作业信号2 | BOOL | MASTER | PLC | 第二工位作业许可，与DBX0.3逻辑相同 |
| DB201.DBX1.2 | 上位机回复完成信号2 | BOOL | MASTER | PLC | 第二工位完成应答，与DBX0.4逻辑相同 |
| DB201.DBX1.3~7 | 预留 | BOOL | — | — | 预留，保持False |

### 5.3 Byte 2~3 — 预留字节

**PLC引用地址：** DB201.DBB2, DB201.DBB3  
全部bit预留，默认值0x00，禁止随意写入。

### 5.4 Byte 4 — 产品类型/配方编号

**PLC引用地址：** DB201.DBB4  
**数据类型：** BYTE（0~255）  
**置位方：** MASTER  
**复位方：** MASTER  

| 值 | 含义 |
|---|---|
| 0x00 | 无效/未定义 |
| 0x01~0xFE | 产品类型编号（具体编码由工艺定义） |
| 0xFF | 保留 |

> MES在写入SN之前，应先写入产品类型/配方编号，确保PLC可以提前准备对应工艺参数。

### 5.5 Byte 5 — 预留字节

**PLC引用地址：** DB201.DBB5，预留，默认0x00。

### 5.6 Byte 6~45 — 产品条码（SN）

**PLC引用地址：** DB201.DBB6 ~ DB201.DBB45  
**数据类型：** CHAR\[40\]（ASCII字符串）  
**最大长度：** 40字符  
**编码规则：** ASCII编码，不足40字节以0x00（NULL）填充  
**置位方：** MASTER  
**复位方：** PLC（流程结束后触发MES清空）  

**适用场景：**
- 产品首次上线（新产品入站）
- 返修产品重新入站
- 转线产品（从其他产线转入）

> ⚠️ **重要：** SN是产品全生命周期追溯的核心标识，MES须确保SN有效且在当前工单中存在，方可发送是否作业信号=True。

### 5.7 Byte 46~55 — 预留WORD区

**PLC引用地址：** DB201.DBW46, DBW48, DBW50, DBW52, DBW54  
**数据类型：** WORD，预留，默认0x0000。

### 5.8 Byte 56~255 — 工位质量数据区

**PLC引用地址：** DB201.DBB56 ~ DB201.DBB255  
**置位方：** PLC（PLC写入检测结果）  
**复位方：** PLC（流程结束后清零）  

地址计算：
- **工位N起始地址：** Byte = 56 + (N-1) × 20
- **工位N结束地址：** Byte = 75 + (N-1) × 20
- **最大支持：** N = 1~10

## 6. PLC→MASTER 信号详细定义（DB200，推荐）

> ⚠️ **说明：** 本章节为推荐补充方案。原始需求仅提供了MASTER→PLC方向，PLC→MASTER方向信号需与PLC工程师协商确认后定版。

### 6.1 Byte 0 — 主状态位字节

**PLC引用地址：** DB200.DBB0  

| 位地址 | 信号名称 | 数据类型 | 置位方 | 复位方 | 说明 |
|---|---|---|---|---|---|
| DB200.DBX0.0 | 心跳 | BOOL | PLC | PLC | 每1秒翻转；MES监测，超过10秒无变化则通信异常报警 |
| DB200.DBX0.1 | 产品到位信号 | BOOL | PLC | MES/PLC | 产品到站，请求MES处理；MES应答后PLC自行复位 |
| DB200.DBX0.2 | 完成信号 | BOOL | PLC | PLC | 工位流程完成；MES回复完成信号后PLC自行复位 |
| DB200.DBX0.3 | 设备故障信号 | BOOL | PLC | PLC | 设备异常标志；MES可触发报警记录 |
| DB200.DBX0.4 | 治具到位信号 | BOOL | PLC | PLC | 治具/夹具到位；可选，视现场定义 |
| DB200.DBX0.5~7 | 预留 | BOOL | — | — | 预留，保持False |

### 6.2 Byte 1 — 工位2扩展状态位字节

| 位地址 | 信号名称 | 数据类型 | 置位方 | 复位方 | 说明 |
|---|---|---|---|---|---|
| DB200.DBX1.0 | 产品到位信号2 | BOOL | PLC | MES/PLC | 第二工位产品到站请求 |
| DB200.DBX1.1 | 完成信号2 | BOOL | PLC | PLC | 第二工位完成信号 |
| DB200.DBX1.2~7 | 预留 | BOOL | — | — | 预留 |

### 6.3 Byte 4 — 当前产品类型

**PLC引用地址：** DB200.DBB4  
PLC上报其识别到的产品类型，MES可用于校验与DB201.DBB4的一致性。

### 6.4 Byte 6~45 — SN（设备读码结果）

**PLC引用地址：** DB200.DBB6 ~ DB200.DBB45  
设备（读码器/RFID）扫描到的产品SN，MES与DB201中写入的SN进行比对校验。

### 6.5 Byte 46~47 — 结果码

**PLC引用地址：** DB200.DBW46  
**数据类型：** WORD  

| 值 | 含义 |
|---|---|
| 0x0000 | 流程正常完成（OK） |
| 0x0001~0xFFFE | 具体NG/异常结果码（由工艺定义） |
| 0xFFFF | 保留 |

### 6.6 Byte 48~49 — 异常码

**PLC引用地址：** DB200.DBW48  
**数据类型：** WORD，设备异常代码（0=正常，非0=具体设备故障编号）

## 7. 握手协议详细规范

### 7.1 产品到位握手（Arrival Handshake）

**触发条件：** PLC检测到产品到位传感器信号  
**参与信号：**

| 步骤 | 操作方 | 信号 | 动作 |
|---|---|---|---|
| 1 | PLC | DB200.DBX0.1 | 置True，通知MES产品到位 |
| 2 | PLC | DB200.DBB6~45 | 写入SN（设备读码结果，若有） |
| 3 | MES | — | 检测到DB200.DBX0.1=True |
| 4 | MES | DB201.DBB4 | 写入产品类型/配方 |
| 5 | MES | DB201.DBB6~45 | 写入SN（工单查询结果） |
| 6 | MES | DB201.DBX0.2 | 置True（回复到位信号） |
| 7 | MES | DB201.DBX0.3 | 置True/False（是否作业） |
| 8 | PLC | — | 读取DB201.DBX0.2和DBX0.3 |
| 9 | PLC | — | 根据是否作业信号决定执行/禁止 |

**超时规范：** MES须在检测到到位信号后 **≤5秒** 内回复，否则PLC触发超时报警。

### 7.2 完成握手（Completion Handshake）

**触发条件：** PLC完成工艺动作  
**参与信号：**

| 步骤 | 操作方 | 信号 | 动作 |
|---|---|---|---|
| 1 | PLC | DB200.DBB56+ | 写入工位质量数据 |
| 2 | PLC | DB200.DBW46 | 写入结果码 |
| 3 | PLC | DB200.DBX0.2 | 置True（完成信号） |
| 4 | MES | — | 检测到DB200.DBX0.2=True |
| 5 | MES | — | 读取质量数据并上传数据库 |
| 6 | MES | DB201.DBX0.4 | 置True（回复完成信号） |
| 7 | PLC | — | 读取DB201.DBX0.4=True |
| 8 | PLC | DB200.DBX0.2 | 复位完成信号=False |
| 9 | PLC | DB200.DBB56+ | 清零质量数据区 |
| 10 | PLC | DB200.DBX0.1 | 复位产品到位信号=False |
| 11 | MES | DB201.DBX0.2~4 | 复位到位/作业/完成回复信号 |

**超时规范：** MES须在检测到完成信号后 **≤5秒** 内回复，否则PLC触发超时报警。

## 8. 心跳机制规范

### 8.1 MES侧心跳实现要求

```
// MES心跳伪代码
Timer heartbeatTimer = new Timer(1000);  // 1秒定时
bool heartbeatBit = false;

heartbeatTimer.Elapsed += () => {
    heartbeatBit = !heartbeatBit;          // 每秒翻转
    WriteToPlc(DB201, offset:0, bit:0, value:heartbeatBit);
};
```

### 8.2 PLC侧心跳监测要求

```
// PLC心跳监测伪代码（梯形图逻辑等效）
IF DB201.DBX0.0 未变化时间 > 10秒 THEN
    触发报警：MES通信异常
    禁止执行新流程
END_IF
```

### 8.3 心跳异常恢复

- 心跳恢复后，PLC自动清除通信异常报警
- MES在心跳恢复后须重新确认初始化状态
- 若心跳中断期间有流程在执行，需人工确认产品状态后再继续

## 9. 初始化规范

### 9.1 MES初始化完成条件

以下所有条件满足后，MES方可将 `DB201.DBX0.1（初始化完成）` 置True：

1. ✅ MES与PLC通信正常（心跳互通）
2. ✅ MES与数据库/质量系统连接正常
3. ✅ 当前工单/配方已正确加载
4. ✅ 上一个产品流程已完整结束（所有握手位已复位）
5. ✅ 设备状态无故障报警

### 9.2 初始化完成信号管理

| 场景 | DB201.DBX0.1 值 |
|---|---|
| 系统启动初期 | False |
| 初始化完成检查通过 | True |
| 流程开始（产品到位应答期间） | True（保持） |
| 流程结束复位期间 | False → 检查 → True |
| 系统故障/通信中断 | False |

## 10. 产品上线/返修/转线处理规范

### 10.1 产品SN校验规则

MES在收到产品到位信号后，依以下顺序进行SN校验：

```
1. 读取SN（DB200.DBB6~45 或 设备读码结果）
2. 查询MES数据库：SN是否存在？
3. 校验当前工单：SN是否在当前工单批次中？
4. 校验工序：当前工位是否为该产品的下一道工序？
5. 校验产品状态：产品是否被锁定/报废/转线限制？
6. 校验配方：产品类型与当前配方是否匹配？
```

### 10.2 上线/返修/转线判断逻辑

| SN状态 | 工序状态 | 判断结果 | 是否作业 |
|---|---|---|---|
| SN存在且在工单中 | 当前为正确工序 | 正常上线 | True |
| SN存在且在工单中 | 已完成该工序 | 重复过站（防错） | False |
| SN存在，返修标记 | 满足返修条件 | 返修允许 | True |
| SN存在，转线标记 | 当前线体为目标线 | 转线允许 | True |
| SN不存在 | — | 未知产品 | False |
| SN存在，已报废 | — | 报废产品，禁止 | False |

### 10.3 是否作业=False时的SN处理

MES须在MES系统中记录该产品的拦截事件，包括：
- SN
- 拦截时间
- 拦截原因
- 工位/设备编号
- 操作员ID（若有）

## 11. 作业许可规范

### 11.1 是否作业信号置True的充要条件

以下所有条件同时满足时，MES方可发送 `是否作业信号=True`：

1. 已发送 `上位机回复到位信号=True`
2. SN校验通过（产品合法且允许在此工位作业）
3. 产品类型/配方匹配
4. MES质量系统/数据库连接正常
5. 无系统级禁止信号

### 11.2 作业禁止处理

MES发送 `是否作业信号=False` 后，须同时：
1. 在MES界面显示禁止原因
2. 记录拦截日志
3. 等待PLC响应（禁止作业报警）
4. 等待人工干预后复位

## 12. 质量数据规范

### 12.1 质量数据结构定义（每组20字节）

| 相对偏移 | 长度 | 数据类型 | PLC变量名（建议） | 说明 |
|:---:|:---:|---|---|---|
| +0 | 2 | WORD | Result_Code | 质量结果码（0=OK，非0=NG） |
| +2 | 4 | REAL | Process_Value_1 | 工艺检测值1（根据工位定义，如扭矩Nm） |
| +6 | 4 | REAL | Process_Value_2 | 工艺检测值2（根据工位定义，如角度°） |
| +10 | 4 | REAL | Process_Value_3 | 工艺检测值3（根据工位定义，如时间s） |
| +14 | 2 | WORD | Error_Code | 详细错误码（0=无错误） |
| +16 | 4 | BYTES | Reserve | 预留字节 |

### 12.2 质量数据上传规范

MES读取质量数据后，须上传以下关联信息：

| 字段 | 来源 |
|---|---|
| SN | DB201.DBB6~45 |
| 工位编号 | 配置定义 |
| 产品类型 | DB201.DBB4 |
| 检测时间 | MES系统时间 |
| 结果码（ResultCode） | 质量数据块+0 |
| 工艺值1~3 | 质量数据块+2~+13 |
| 错误码（ErrorCode） | 质量数据块+14 |
| 操作结论（OK/NG） | ResultCode==0 ? OK : NG |

### 12.3 质量数据复位规范

- **PLC负责：** 在收到 `上位机回复完成信号=True` 后，将对应工位质量数据块清零（写0x00）
- **MES负责：** 在读取质量数据并确认上传成功后，方可发送 `上位机回复完成信号=True`
- **禁止：** MES未上传成功前不得发送完成信号，避免数据丢失

## 13. 超时与异常处理规范

### 13.1 超时阈值定义

| 超时场景 | 建议阈值 | 超时触发方 | 处理动作 |
|---|---|---|---|
| 心跳超时 | 10秒 | 对方 | 触发通信异常报警，停止流程 |
| MES回复到位信号超时 | 5秒 | PLC | PLC触发"等待MES到位应答超时"报警 |
| MES回复完成信号超时 | 5秒 | PLC | PLC触发"等待MES完成应答超时"报警 |
| PLC完成信号超时 | 工艺时间+30秒 | MES | MES触发"PLC完成超时"报警 |
| 质量数据读取超时 | 3秒 | MES | MES重试3次，失败后记录异常并报警 |

### 13.2 异常报警处理优先级

| 级别 | 报警类型 | 处理要求 |
|---|---|---|
| P1（紧急） | 心跳超时 | 立即停线，通知班组长 |
| P2（重要） | 产品拦截（不可作业） | 操作员处理后可继续 |
| P3（一般） | MES/PLC应答超时 | 系统自动重试，超限后人工介入 |
| P4（提示） | 质量数据NG | 记录并转入NG处理流程 |

### 13.3 通信中断恢复规范

1. 中断期间：PLC停止接受新产品，保持当前安全状态
2. 恢复后：MES主动检查并同步PLC当前状态
3. 恢复后：若PLC有未完成的握手，MES须根据状态决定继续或复位
4. 恢复后：MES重新发送 `初始化完成=True` 表示就绪

## 14. 复位规范

### 14.1 流程正常结束复位顺序

```
步骤 1: PLC 置位 完成信号=True，写入质量数据
步骤 2: MES 读取质量数据
步骤 3: MES 上传质量数据到数据库
步骤 4: MES 置位 上位机回复完成信号=True
步骤 5: PLC 检测到 上位机回复完成信号=True
步骤 6: PLC 复位 完成信号=False
步骤 7: PLC 清零 质量数据区
步骤 8: PLC 复位 产品到位信号=False
步骤 9: MES 检测到 PLC产品到位信号=False（或超时后主动复位）
步骤10: MES 复位 上位机回复到位信号=False
步骤11: MES 复位 是否作业信号=False
步骤12: MES 复位 上位机回复完成信号=False
步骤13: MES 清空 SN字段
步骤14: MES 确认所有握手信号已复位
步骤15: MES 置位 初始化完成=True，等待下一产品
```

### 14.2 异常中断复位规范

- 人工复位时，按照正常复位顺序逐步操作
- 不允许跳过步骤或直接强制复位所有信号（可能导致数据丢失）
- 若质量数据未上传成功，须人工确认后方可复位

## 15. 接口实现要求

### 15.1 MES软件实现要求

| 要求 | 说明 |
|---|---|
| 通信轮询周期 | ≤ 100ms（建议50ms） |
| 心跳更新精度 | ±200ms（目标1000ms更新一次） |
| 数据读写原子性 | 同一DB块的相关数据须在同一通信帧中读写 |
| 异常日志 | 所有握手超时、拦截事件须记录到MES日志 |
| 多线程安全 | 通信线程与业务线程须加锁保护共享数据 |
| 断线重连 | 支持自动重连，重连后主动同步状态 |

### 15.2 PLC程序实现要求

| 要求 | 说明 |
|---|---|
| 心跳监测 | 每扫描周期检测MES心跳变化，超10秒触发报警 |
| 握手超时 | 所有等待MES应答的步骤须有超时保护 |
| 质量数据写入 | 完成信号置位前须确保质量数据已完整写入 |
| 复位逻辑 | 收到MES回复完成信号后方可复位，不允许提前复位 |
| 上电初始化 | PLC上电后须将所有DB数据块清零，再等待MES初始化 |
